经验与能力不能等同

2020游戏策划就业形势分析：

消极的直观感受：在游戏策划领域，20届与21届撞车明显，本科生与硕士生同台竞技，内卷现象严重；秋招时间错位，网申在20年暑假就开始了；信息差效应，大厂hc越来越多偏向高学历人才，高学历人才流向大厂而其他人不被看重，岗位需求逐渐偏向大厂，将逐渐加高行业门槛形成循环；甚至于这个门槛即使空有经验、没有能力，想从小厂跳到大厂也会变成隐形的壁垒

积极的直观感受：大厂hc（编制）比较多，如网易有1000+，腾讯有3000+，字节跳动有10000左右；薪资倒挂，大厂校招比往年校招、普通社招薪资待遇都要高很多，比如网易校招硕士开到1.8k×16，而网易社招硕士开到1.2k×14；项目类型多样化，包括二次元项目、乙女游戏、中重度游戏（包括MMO、FPS、SLG、放置挂机等）休闲游戏、教育游戏等；社会舆论氛围缓和、包容。

常见问题的心理准备：

做好心理建设：

  1. 就业形势严峻

  2. 挫折感比较大

  3. 一定要正视求职就业的问题:

    3.1. 通过数据分析清晰冷静面对自己所处的位置

    3.2. 然后根据自身情况和目标制定学习计划和行动计划

  4. 冷静对待现状，冷静分析问题，制定对应对策

同时一定要恢复自信，因为有：

  1. 国家政策鼓励
  2. 疫情利好因素的影响
  3. 行业数据可观
  4. 每个人都应该有自己的自信
  5. 当前专业不限，学历要求还较低
  6. 更加看重能力，而非学历
  7. 要有自我认可、自我鼓励、对未来的信心

另外要做好自身的能力考核：

  1. 经验不等于能力。即使空有经验、没有能力，想从小厂跳到大厂也会存在隐形的壁垒

  2. 能力是更隐性的存在

  3. 能力模型：

    3.1. 综合素质，包括逻辑思维能力、创新能力、学习提炼能力、沟通能力/团队合作、执行力、AQ/EQ（心态/胸怀）

    3.2. 专业知识，包括基础知识、专业素质、专业知识

    3.3. 用户需求导向，包括用户需求理解、用户管理与服务、市场与用户调研分析

    3.4. 领导管理能力，包括项目管理、人才培养、知识传递、战略分析、市场方向把控

  4. 第二技能，比如会点美术的策划、懂点U3D的文案、会些编程的数值

  5. 求职作品也是能力的体现，比如策划案、拆解案例、可玩性高的demo

二、自荐信和标准简历

简历的准备：

面试环节中，简历还重要吗？简历的作用在于包装求职者的人设，作用在于通过hr的筛选

至少至少要有：简历、简历第二页、自荐信三部分

标准格式：

  1. 基本信息，包括姓名、性别、头像、籍贯等

  2. 个人自我评价，即自我认知，不要与其他部分重复，不要太复杂、啰嗦，避免挖坑，字数尽量控制在150字内（25*6左右）。自我评价要记熟、记牢，在面试中口语化。如图：

    本人修改后的自我评价：

    愿意倾听并乐于协作，在团队中乐于与他人交流合作，使工作更加高效、全面；

    喜欢将想法实践落地。当一个想法可行且有价值，经过与团队内部的深入沟通后，我会着手将其实现；

    热爱学习且求知欲强。作为游戏文案策划，不仅需要有丰富的知识涉猎，也需要掌握足够多的工作技能。利用课余时间通过与他人交流学习、了解游戏作品及书籍，不断学习“剧情设计”、“游戏设计”和“用户体验”的相关知识。

  3. 教育经历，包括学校（可以由被烛或介绍）、专业、（重点的）主修课程、就读时间、学历学位等

  4. 校园经历，包括社团经历、社会实践、在校期间的实习、比赛或项目经历

  5. 工作经验：

    5.1. 如果是转行的则适当简约，尽量争取寻找出一些共通性体现自己的能力；

    5.2. 如果是转岗的则表现出游戏从业经历，可以重点描述个人项目经验、自己的理解感悟，简要描述承担的工作内容

  6. 技能特长（简要描述）

  7. 兴趣爱好（简要描述）

注：如果相关经历较少，可以附上自己的求职作品；或者先找小厂实习再备战春招

基本要求：

  1. 排版工整，字号最小五号字，使用pdf文件格式
  2. 言简意赅，避免口语化，不要太罗嗦，避免内容重复
  3. 关键信息突出，目标感明确
  4. 简历从头到尾的核心意图要明确，应聘文案要有文案的写法
  5. 模块分清，可阅读性更高
  6. 游戏经历挑几款自己真正熟悉的游戏写

常见问题：

  1. 自我评价混杂个人爱好及游戏经历
  2. 不要使用底图模版及水印
  3. word文件的排版
  4. 把实习经历写成工作经历
  5. 把自我评价写成个人总结
  6. 缺少内容，如兴趣爱好、校园经历、自我评价、游戏经历、专业及专业课程等
  7. 基础信息过于简单
  8. 自我评价放在后边
  9. 内容之间留白过多
  10. 不建议写太多游戏经历
  11. 求职意向太泛，目标感不明确
  12. 画蛇添足，无关经历描述过多
  13. 口语化严重
  14. 不要用生活照作为头像
  15. 词不达意
  16. 不会主动向游戏行业靠拢
  17. 游戏理解和分析较浅
  18. 把简历和求职作品放在同一文件内
  19. 三个月内的无效工作经历（实习除外）

项目经历STAR法则

S情景、T目标、A行动、R结果

项目经历一定要清楚表达。不仅是在简历中，也包括在面试环节要能够将项目经历表达清楚，否则容易被怀疑造假。

重点描述经验，简要描述自己的工作内容

将在校园内取得的经历放在校园经历内，不要在奖项内容单独写

简历第二页：

简历第二页应该包括：

  1. 游戏经历概述。重点突出12个类型，每种列举12个真正熟悉的游戏
  2. 游戏理解与分析
  3. 某游戏拆解案概要

游戏经历概述：

游戏经历要丰富——游戏是游戏策划的沟通语言，可以提高沟通效率、引发具象化联想

如何正确地写自己的游戏经历？

  1. 保持精简，不要长篇累牍

  2. 谦虚谨慎。不要使用太夸张的话术，诸如“十多年”、“几万小时时长”、“游玩了上千款游戏”、“非常熟悉”、“善于分析”、“对……理解深刻”之类慎用

  3. 除非面试岗位有要求，不要以表格形式列举，尽量用文字概述，要突出特色、时间长度

  4. 注意不要列举太多游戏：

    4.1. 不要太强调个人喜好

    4.2. 多玩主流热门的游戏

    4.3. 多积累游戏上手体验，不要总是云游戏

    4.4. 去掉年代久远的游戏

    4.5. 去掉一些自己不是那么熟悉、容易问倒的游戏

    4.6. 去掉小众游戏

    4.7. 不要强行填充一些游戏假装丰富

  • SLG类：率土之滨
  • MMO类：天涯明月刀手游、逆水寒、一梦江湖、天谕手游
  • 卡牌类：刀塔传奇、公主链接

游戏理解与分析：

包括对游戏经历中某款游戏的评测或拆解。篇幅应在半页word左右，控制字数在600字左右。

结构内容：游戏理解与分析中，需要总结某款游戏的基本状况简述、设计目的、优缺点、改善建议

优缺点：优点提炼3点、缺点1~2个。可以围绕“玩家体验”、“设计者”、“市场运营”多个角度展开描述

改善建议需要针对缺点改善、提出合理设计方案，并有明确的改善问题及目标。建议方案中应该涉及系统定位、设计目的、用户体验、系统介绍四个部分

建议：

  1. 要保证观点清晰、逻辑明确、论据论述充分、总结陈述有力

  2. 不必千篇一律，核心在于写出深度和专业性，站在设计者角度、用户体验、市场运营等角度看问题

    可以只针对某一系统撰写，也可以写对游戏行业的理解，也可以写某乙女游戏的人物感情线，可以写某MMO游戏的氪金付费系统，或者写写魔兽世界的圆桌理论，或者某开放世界的探索体验。

  3. 尽量使用专业术语，减少口语化表达

  4. 多隐含一些设计思维

  5. 多用数据说话

  6. 多做横向对比，做竞品分析

作用：

  1. 引导面试官去问自己有所准备的游戏
  2. 丰富简历
  3. 表示自己对某品类有深入了解

求职作品的准备：

  1. 系统策划案

  2. 系统拆解

    系统拆解可以拆整个游戏，也可以拆游戏中的某个系统，有时候单拆游戏某个系统效果更好。推荐拆游戏的核心玩法或核心系统（特色系统）或付费系统，也可以拆解文案系统

  3. 反策划案。（反策划案约等于系统设计案，前者是用了别人游戏的截图和设定，后者基于自己的需求做的系统设计文档）

自荐信：

作用：表明自己的诚恳态度，适合于投自己非常想去的公司

一般来说，在电子版中自荐信和简历是分开的；在纸质版文档中，自荐信在第一页，简历在第二页

建议自荐信有两个版本，通用版本和有针对性的专属版本（表达真情实意）。前者适合海投，后者在面试非常心仪的公司时使用

写法建议：

  1. 办公文书的方式，稍微正式严肃，多使用敬语
  2. 不建议过于传统，稍微互联网化一些
  3. 多用比较有温度、感性的语言
  4. word篇幅在一页以内
  5. 采用三段式，三个段落构成

  6. 落款及时间

  3. 讲一讲你最喜欢的游戏
  4. 你对某游戏的玩法怎么看

面试官旁敲侧击要尽可能地如实回答；或者换个概念，如“最近几个月……”、“最近一年……”。总之尽量避免提及自己不熟的游戏。

通常来讲，一场面试一般会聊12款游戏，一般就问4个大问题，每个大问题来回追问34次，一共1216个小问题；每个小问题23分钟，一场面试下来一共30~45分钟左右

对游戏产业的认知：

建议：

  1. 多看新闻
  2. 多看资讯
  3. 多了解行业数据
  4. 关注热点话题

游戏理解能力：

观点类问题：

对某游戏怎么看？某款游戏哪里好玩？为什么喜欢？优缺点分析？

  1. 讲特色讲亮点讲用户喜好
  2. 带一点个人观点和判断
  3. 不要太口水、啰嗦
  4. 不要过多描述细节

设计类问题：

设计一个关卡/英雄/玩法/副本？

标准格式：

  1. 系统定位
  2. 设计目的
  3. 用户体验
  4. 功能点。举例：某个功能怎么设计？具体流程？原型图？规则逻辑？（逻辑图与流程图不同，逻辑图给程序看，流程逻辑更详细）

↑注意标准格式，方法论和思维清晰

  3. 反问：

    3.1. 这样回答可以吗

    3.3. 请给予指点

职业规划：

3年后你希望自己做什么职位？3~5年→主策

说说未来3年的想法？未来的规划？

从执行策划做起，逐渐到达XX策划职位，另外多学点其他的东西，比如……，未来希望能做制作人

  1. 为什么做游戏？
  2. 为什么不去其他岗位？
  3. 软件工程为什么不去做编程？
  4. 为什么转岗？之前为什么不做策划？

都应该回答“因为喜欢游戏”，但要显出自己的不同

提炼好这些问题的答案，记在本子上背下来准备随时被拷问

反问环节：

雷区

  2. 新人是否包食宿
  3. 公司其他福利
  4. 不熟悉公司的游戏类型。如果被问到时，建议回答时职业化一点：不要过分强调自己的个人喜好，要跟着公司项目走，我是来应聘游戏策划，但不是来应聘我喜欢游戏类型的游戏策划
  5. KPI相关的问题暂时不要问

加班问题必须在求职前考虑。加班可以接受，根据项目需要、服从公司安排

反问问题时应注意以下几点：

  1. 如果被问到什么时候可以入职，需要如实回答

  2. 反问环节要表现出对公司的渴望、对工作岗位的热爱

  3. 反问环节要表现出低姿态和对新岗位的渴望

    3.1. 求职意愿强烈

    3.2. 入职意愿强烈

    3.3. 对公司认同感高

  4. 雷区问题尽量不要提及

    3.6. 语言表述准确

四、游戏拆解讲解：

游戏拆解是游戏从业者的通用能力，是几乎所有游戏策划、运营、发行岗的通用能力，是必备的万金油技能

案例：

游戏拆解的意义和价值：

  1. 求职作品
  2. 竞品分析
  3. 市场运营
  4. 反策划
  5. 项目立项
  6. 拆得越深，对游戏的理解程度越深，才能做好游戏

游戏拆解的目的：

  1. 深入了解和剖析游戏产品
  2. 为游戏分析作基础信息收集
  3. 为游戏反策划做基本功课

内容：

用思维导图的形式来做，但最终成型是一份报告（思维导图+分析方案word或者ppt）

误区：

  1. 大篇幅写很多游戏的功能或玩法介绍
  2. 使用大量截图
  3. 把游戏理解分析写成了游戏说明书

拆解方法：

  1. 功能划分：先穷举打草稿，其次归类。注意排版
  2. 按系统的定位、作用、价值等进行分类
  3. 按一定规律排序。按角色游玩的先后顺序排序，或者按界面功能模块排序
  4. 相对而言只需要拆解到功能点即可，不必拆详细规则。游戏拆解能让人对系统有整体理解即可，而反策划需要详尽到每个功能点的设计规则和每个参数设定
  5. 给出一个合适的结论，输出一份报告

注意：

  1. 理清游戏核心玩法

  2. 分析游戏核心定位、类型

  3. 对游戏得出一个清晰明确的脉络

  4. 进行系统框架图的设计时，核心在于分清主次、位置、关联关系。系统框架图中，哪个系统的连接线比较多就说明那个系统比较核心。如例：

  5. 进行资源关系图的设计时，如例：

思维导图制作：

  1. 在功能分类的基础上展开1~2层就够了，有需要再展开
  2. 按一定规律排序，一目了然
  3. 注意排版
  4. 不要把思维导图当作word用
  5. 控制单个主题的字数，不要太长
  6. 不要拉连接线
  7. 不要加批注
  8. 不建议太多层级

分析方案的形式：

分析方案的形式是思维导图截图+word文字。

要注意：仅有一张思维导图远远不够，拆而不解没有意义，看不出什么能力；核心在于为什么要这么做，word就是这份工作。

拆解分析的重点在于分析清楚游戏的特色系统分析及功能说明、优缺点、改善建议、总结陈词等；具体内容见“系统分析”部分。不建议每个系统都写一遍，控制好篇幅和字数。

案例：

如果两个系统之间存在联系，但又不能直接写在一起。如果非常重要的话，首先应该尽量写在同一个模块，其次在word里说明；或者用visio重新作系统框架图连线

系统分析：

写在word里，不要写在思维导图里。具体内容包括：

系统定位：

资源产出要有主次之分。比如升级经验的产出要有主要产出系统、次要产出系统、若干小比例产出。要写明这个系统是哪些资源的产出系统

设计目的：

  1. 新手引导
  2. 成长助力
  3. 付费转化
  4. 激化玩家间矛盾
  5. 诱惑核心玩家付费
  6. 促进高R玩家转化
  7. 提升用户在线时长
  8. 提升用户活跃度
  9. 提升用户粘性
  10. 提升用户交互
  11. 激励刺激，奖励激励……等

资源流向：

  1. 有消耗必须有产出
  2. 每个系统都有相互之间的定位和对应关系
  3. 比较普通的资源会对应多个系统，而高级资源一般只对应于一个系统和付费

玩家体验

  1. 低难度
  2. 中等难度
  3. 高难度
  4. 比较舒畅的
  5. 有明显卡点的

用户分流

不要让用户挤在一个系统，不要让用户行为过于单一

时间分配

  1. 同一资源的养成策略：
  2. 优先玩主要系统
  3. 其次玩次要系统
  4. 最后选择性体验或者放弃其他系统

用户粘性

  1. 提升用户层次感
  2. 提升用户幸福感
  3. 提升用户荣誉感
  4. 提升用户成就感
  5. 提升用户满足感
  6. 提升用户新鲜感
  7. 让用户感觉赏心悦目→音乐、美术等

运营数据提升

留存、付费、up池、LTV

能力考查：

  1. 综合能力
  2. 系统思维
  3. 思维逻辑
  4. 用户心理

最后总结

以系统分析为基础、以能力作为支持，最后得出结论。拆解报告的文字占据篇幅1页即可。其中要包括系统分析、功能说明、改善建议、总结陈词等。截图插入word，最后把大的思维导图作为附件

  1. 站在市场角度的核心亮点
  2. 这个游戏的核心系统玩法，提练出最吸引人的地方/最好玩的地方
  3. 优缺点、改善建议

五、新人如何面试？

面试失败常见的问题原因：

  1. 简历质量
  2. 心态问题：自我感觉良好。建议提前准备，自身找原因
  3. 执着于岗位
  4. 没有复盘。建议书面复盘，整理题库，优化改善答案，形成强记忆、对答如流
  5. 个人习惯问题，如脑与嘴不同步、紧张情绪、说话啰嗦、语言提炼缺乏逻辑条理、抓不住重点

面试准备

  1. 环境：给面试官一个比较开阔的视野，明亮、不刺眼的光线，环境不要噪杂（没有噪音、没有人来人往）
  2. 视线问题：不要一直盯着面试官看，可以时不时看一眼作为礼貌性回应或者响应式回答；可以看着面试官鼻尖以下的三角区域。注意保持微笑，自然放松
  3. 设备问题：建议使用手机，调整好角度
  4. 着装。注意着装不要太随意
  5. 仪表仪态。面容整洁，注意毛发、衣服清洁，保持精气神，不要熬夜，不要剧烈运动。注意站姿坐姿保持得体端正。尽量保持职场的状态
  6. 言行举止。注意说话时的语言习惯以及肢体语言习惯，保持眼神交流和微笑
  7. 简历。线下面自带一份简历，防止面试官忘带，简历有更新可以带着

准备求职作品时，系统拆解一定要注意思维逻辑和系统分析。

新人如何作自我介绍：

自我介绍的意义：

  1. 面试官用来开场，希望通过应聘者最熟悉的内容缓和气氛
  2. 好的开场取决于第一个话题，会影响正常面试的氛围。应聘者的性格特征、语气语调会影响面试官的面试节奏，面试环节中要保持放松的语气

自我介绍基本公式

大致可分为有四个结构：

  1. 我是谁、来自哪里
  2. 自我评价，或者表达自己
  3. 我能创造什么价值，或者我来做什么
  4. 最后总结，表达期盼

结构案例：

其中2段式比较适合比较自我、内向一点的同学，表现比较中规中矩；3段式适合比较会自夸、善于表现的人；4段式适合逻辑能力比较好、思维清晰、善于把握技巧的同学，表现比较理性

自我介绍后面试官没有接话时，要自己主动寻找话题：

  1. 开始谈自己的项目经历
  2. 开始谈自己的校园经历
  3. 开始谈自己的游戏经历

争取引导面试官话题：

在自我介绍里隐藏一些面试官可能感兴趣的话题；并且前提是心里已经有备好的答案，提前做好应对方案

建议：

  1. 选择一个适合自己的方式
  2. 准备背诵稿
  3. 准备一个备选方案，应对意外情况

常见问题：

  1. 背稿雷同

  2. 一口气念简历，面试官很想打断甚至会走神

  3. 没有给面试官留余地，没有话题点。应该主动帮面试官引出话题点

  4. 不知道怎么开始

    4.1. 语气词过多

    4.2. 准备时间过长

  5. 说的短，缺漏内容

  6. 不懂临场应变

    6.1. 面试官不耐烦

    6.2. 面试官比较仓促

    6.3. 面试官催得急

  7. 怀疑地位不对等。面试官眼中大家都是平等的，目的是寻找自己的同事

面试技巧

电话面:

如果有复盘记录，如果有简历+求职作品，如果有整理笔记，做好现场应对；不要念稿，不要走神，不要停顿太久；自然反应，按自己的理解回答。谨慎小动作

视频面：

如果有复盘记录，如果有简历+求职作品，如果有整理笔记，做好现场应对；不要念稿，不要走神，不要停顿太久，眼睛不要瞟来瞟去；自然反应，按自己的理解回答。更加谨慎小动作

线下面：

  1. 无论什么情况下，找一个不要朝外的座位避免受到会议室之外的东西影响，也会有专业的HR安排座位
  2. 保持固定坐姿，不要晃动、不要有小动作，不要跷二郎腿
  3. 聚精会神。保持良好状态，保持思路连贯
  4. 尽量一直朝着面试官，保持微笑，保持良好沟通
  5. 可以在眼前放一份简历，需要时可以瞟一眼，或者提醒面试官简历有什么内容

常见问题

简历上应注意的问题：

  1. 基本信息很少
  2. 自我评价写得不够好
  3. 教育经历没有写全
  4. 校园经历/实习经历没有用STAR法则来写
  5. 项目经历尽量要包装到与游戏相关，尽量使用产品经理的思维来写
  6. 工作经历尽量包装到一些能力能够在游戏行业使用；如果完全无关可以简单概括
  7. 技能和兴趣爱好的内容不要与自我评价的部分相混，不要和游戏经历相混

面试问题：

感觉被面试官挑战时：

  1. 你有什么缺点？可以回答一些无关痛痒、无伤大雅的缺点或者不那么明显的劣势，委婉回答、先抑后扬，比如“我的性格有点内向但我喜欢思考”
  2. 你有什么优势？根据自我评价来写，不要表达过分的自信

为了防止忘词、大量空白，话可以说慢一点：

  1. 开口前先打好腹稿，不要一开始使用确定的数
  2. 在心里默念数字123，甚至可以掰着手指列出条理

现场提了不该提的话题，自己给自己挖坑。建议：

  1. 抓住重点，不要踩错边界
  2. 抱紧自己熟悉的游戏

建议在面试过程中，控制和管理自己的情绪，控制和管理自己的口头表达，不要在面试中出现慌乱

反问环节：

反问环节可以问什么问题：

  1. 如果进展比较顺利：

    1.1. 可以问什么时候会入职

    1.2. 会安排到哪个项目

    表现出急于求职的心态

  2. 如果进展不是特别顺利：

    可以询问评价或改善意见

不要踩雷：

  1. 不要直接问薪资
  2. 不要直接问薪酬福利待遇
  3. 不要问能否不要996或加班费
  4. 不要急于了解公司内部信息

专业问题：

  1. 行业认知
  2. 游戏理解
  3. 系统分析
  4. 游戏设计
  5. 关卡设计
  6. 战斗系统
  7. 数值系统
  8. 文案世界观
  9. 剧情设计
  10. 运营数据
  11. 行业热点新闻
  12. 热门数据

总结：

  1. 面试是一个综合的过程
  2. 有很多注意事项，也有很多现场变故处理
  3. 需要使用一些语言技巧和沟通技巧
  4. 找到适合自己的才是最好的
